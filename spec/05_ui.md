# UI Modes Breakdown

## Mode 1 - Floating Widget
- **Scope**: Live control surface during practice.
- **Layout**: Small, always-on-top overlay with minimal footprint.
- **Components**:
  - Live Session timer readout
  - Live Note timer readout
  - Recording/Listening status
  - Session Start/Stop control
  - Sensitivity slider
  - Session log list
  - Dashboard launch action

## Mode 1b - Floating Widget Recovery
- The Dashboard must expose a direct, always-visible action that opens or focuses the Floating Widget.
- The recovery action must restore a closed widget if needed, not only focus an already-open window.

## Mode 2 - Dashboard Window
- **Scope**: Historical intelligence and session management.
- **Layout**: Full window, resizable, tab-based navigation.
- **Tabs**:
  - **Summary**: Metrics and pure CSS bar charts
  - **Calendar**: Month grid with click-to-inspect day details and zoomable day timelines
  - **Sessions**: Session list with editable session names, expandable note details, and note-level metadata editing
  - **Settings**: Persisted preferences for the Floating Widget, including opacity control

## Dashboard Detail Surfaces
- The Sessions tab must render each session as a collapsible card so note details are hidden until the user expands the session.
- The expanded session surface must expose note label, tags, description, start time, note duration, and the derived active duration for the parent session.
- The Calendar day panel must provide zoom-in and zoom-out controls so the user can inspect session start and end times at a finer visual scale.
- The Calendar day panel must surface total session duration and total active note duration for the selected day.
- The Summary tab must surface a comparison between total session duration and total active note duration, plus the resulting gap time.

## Settings Surface
- The Settings tab must present the Floating Widget opacity as a slider or equivalent progressive control.
- The control must persist the preference and apply the change immediately to the live Floating Widget.

## Separation Rules
- Floating Widget must remain minimal and responsive.
- Analytics and calendar rendering belong only to Dashboard Mode.
- Floating Widget opacity is controlled from Dashboard Mode, but the resulting visual state belongs to the Floating Widget window.
- Dashboard rendering must not degrade floating widget timer responsiveness.
